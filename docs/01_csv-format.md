## 1. ファイル基本仕様

| 項目 | 仕様 | 備考 |
|:---|:---|:---|
| **文字コード** | Shift-JIS (CP932) | **最重要**: 日本語環境での互換性を優先 |
| **改行コード** | **CRLF (0x0D, 0x0A)** | **最重要**: すべての行末（ヘッダー含む）に必須 |
| **区切り文字** | カンマ (,) | 列の区切り |
| **引用符 (")** | 付与する列は行の位置と列で異なる | 詳細は「6. 引用符の付与ルール」を参照 |
| 1行目 | 列名ヘッダ | 全列名がダブルクォーテーションで囲まれてカンマ区切り |
| 2行目 | 工事概要（タイトルやプロジェクト情報） | |
| 3行目以降 | 見積内容（項目名、数量、単価、金額、計算式など） | |

## 2. 階層構造とレコード配置

本システムでは、物理的な行順と階層フラグの組み合わせでレコードが連続した「シート」とその「階層構造」を表現する。

### 階層フラグの定義

| フラグ | 意味 | 補足 |
|:---|:---|:---|
| `-0` | ルート階層 | ファイル全体のタイトル。レコードの2行目 |
| `1` | トップ階層（親項目） | 見積の大きな括り（工事区分など） |
| `-1` | 内訳開始ヘッダ | 直上の階層1項目に対する内訳シートの開始宣言 |
| `2` | 内訳明細（2階層目） | 階層-1の直後に配置される具体的なデータ行 |
| `-2` | 2階層目の内訳開始 | |
| `3` | 内訳明細（3階層目） | 階層-2の直後に配置される |
| `-3` | 3階層目の内訳開始 | |
| 以降同様 | 最大10階層まで | |

### レコード配置ルール（幅優先 ─ 最重要ルール）

**原則: 同じ親を持つ同じ階層のレコードは必ず連続して配置する。**

具体的な配置手順:
1. `階層-0` を先頭に配置（1行だけ）
2. `階層1` の項目を **すべて連続して** 配置（トップ階層シート）
3. 各階層1項目に対応する `階層-1` と `階層2` のブロックを順次配置
4. 階層2に子がある場合、**同じ親の階層2を全て配置してから** `階層-2` と `階層3` を配置
5. データ生成時には必ず **同じ親の兄弟が連続配置されているか** チェックする

### 配置の正しい例（幅優先）

```
-1, 00001, 【基本数量】       ← 親階層の開始
 2, 00003, 諸元               ← 子1
 2, 00004, 取引先別単価       ← 子2（兄弟を先に連続させる）
-2, 00003, 諸元               ← 子1の中身の開始
 3, 00005, 1F床面積
-2, 00004, 取引先別単価       ← 子2の中身の開始
 3, 00006, ﾍﾞﾀ基礎[001]
```

### 階層構造の図解

```
[階層-0] (ID:00000) プロジェクト情報
 ┣ [階層 1] (ID:00001) 仮設工事       ← すべての階層1を先に連続させる
 ┣ [階層 1] (ID:00002) 基礎工事
 ┗ [階層 1] (ID:00003) 木工事
[階層-1] (ID:00001) 仮設工事 内訳開始  ← 親(階層1)と同じIDを使う
 ┣ [階層 2] (ID:00004) 仮設詳細       ← 新しいIDを使う
 ┗ [階層 2] (ID:00005) 外部足場       ← 同じ親の兄弟は連続配置
[階層-2] (ID:00004) 仮設詳細 内訳開始  ← 親(階層2)と同じIDを使う
 ┣ [階層 3] (ID:00006) ＸＸＸＸ       ← 新しいIDを使う
 ┗ [階層 3] (ID:00007) ＹＹＹＹ
[階層-1] (ID:00002) 基礎工事 内訳開始  ← 兄弟階層
 ┣ [階層 2] (ID:00008) べた基礎底盤
 ┗ [階層 2] (ID:00009) べた基礎立上
...続く
```

## 3. ID管理ルール

- **書式**: 5桁のゼロ埋め文字列（例: `00001`）
- **性質**: ファイル内でユニーク（一意）であること
- **親子紐付け**: `階層-N` のレコードは対応する `階層N` と同じIDを保持する。これが親子関係の紐付けキーとなる
- **注意**: IDは行の連番ではない。階層Nとその内訳（階層-Nおよび子階層）のグループを紐付ける共通キー

## 4. 行属性（フラグ集約列）

21桁の各文字が特定の機能をON（文字あり）／OFF（`0`）で制御する。
**「シート」とは同じ階層の行のかたまりのこと。**

| 位置 | 文字 | スイッチ名 | 金額加算 | 機能概要 |
|:---|:---|:---|:---|:---|
| 1 | **B** | 歩掛行 | | 下階層の合計を単価にセットする |
| 2 | **C** | コメント行 | 禁止 | 項目名～備考フィールドに跨ってコメント文字列あり |
| 3 | **T** | 小計行 | 禁止 | シート先頭～直前までの合計 |
| 4 | **N** | 値引行 | | 値引を明示的にしているマーク |
| 5 | **H** | 非加算行 | 禁止 | シートの合計に含めない |
| 6 | **K** | 計算行 | | 計算式が組み込まれた行 |
| 7 | **Z** | 非課税行 | | 消費税の計算対象としない |
| 8 | **R** | 合計対象行 | | SUM()関数の対象。マークが無ければ全行が対象 |
| 9 | **L** | 合計範囲境界 | | SUM()関数の範囲をリセットする境界 |
| 10 | **X** | 非印刷行 | 禁止 | 加算も印刷もしない。作成者コメント用 |
| 11 | **M** | 非加算計算行記憶 | 禁止 | 非加算計算行であったことの記憶 |
| 12 | **S** | 基本数量行 | 禁止 | 下階層は変数・値を格納した行の集まり |
| 13 | **Y** | 消費税行 | 禁止 | 直前までの合計から消費税を計算 |
| 14 | **I** | 印刷明細行 | | この下階層は印刷しない |
| 15 | **A** | 請求行 | 禁止 | 請求書の今回請求額・入金済額 |
| 16 | **F** | 数量計算式有 | | 数量に計算式が設定されている |
| 17 | **F** | 単価計算式有 | | 単価に計算式が設定されている |
| 18 | **F** | 単位量計算式有 | | 単位量に計算式が設定されている |
| 19 | **D** | 改行マーク | | 印刷時にこの行の次で改ページ |
| 20 | **I** | 変更不可マーク | | シート入力時に変更できない列 |
| 21 | **O** | 展開印刷行 | | 印刷時に直後に下階層を続けて印刷（直下1階層のみ） |

### コメント行（Cフラグ）の特殊ルール

- **1行コメント**: 「番号」列に ` C`（半角スペース1つ＋大文字C）を必ず入力
- **複数行コメント**: 「番号」列に `L5` 等（`L`＋印刷時の行数）を入力。`L5`は5行分のエリア確保

## 5. 計算ロジックと基本数量シート

### 基本数量シート (Basic Quantity Sheet)

行属性に「S」フラグを持つ階層1項目の下階層（階層2）を指す。

- **変数定義**: 項目名に記述された文字列が「変数名」、「予算数量」がその値
- **計算式参照**: 見積シート内の計算式列から、これらの変数名を直接参照可能
- **詳細**: 計算式の詳細な文法は `02_formula-guide.md` を参照

### 計算式の基本書式

| パターン | 例 | 意味 |
|:---|:---|:---|
| `=右辺式` | `=延床面積*0.15` | 予算数量に代入 |
| `代入先=右辺式` | `予算単価=2300*0.7` | 予算数量以外に代入 |

## 6. 引用符 (")の付与ルール

### 1行目（ヘッダ行）
すべての列名にダブルクォーテーションを付与する。

### 2行目（工事概要行）
1行目の列項目とは無関係。サンプルの形式で出力する。

### 3行目以降（データ行）

**ダブルクォーテーションで囲む列:**
番号, 項目名, 材質・寸法, 予算単位, 見積単位, 備考, 取引先ｺｰﾄﾞ, 取引先名, 原価ｺｰﾄﾞ, 原価名, 種別ｺｰﾄﾞ, 種別名, 出典根拠, 分類名, 部分名, 計算式, 歩掛/単位

**囲まない列（数値・フラグ列）:**
階層, ＩＤ, 行属性, 予算数量, 予算単価, 見積数量, 区分, 掛率 等

## 7. ファイル出力時の厳格な命令

本システムのインポートエンジンは、行末が **CRLF (0x0D, 0x0A)** の場合のみレコードの区切りを正しく認識できる。

- **すべての行（最終行含む）に CRLF を付加すること**
- **文字コードは必ず Shift-JIS (CP932)**
- UTF-8やBOM付きで出力するとインポート不可
