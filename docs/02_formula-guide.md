## 1. 計算式の基本構造とルール

計算式は「変数への代入」か「結果を返す」のどちらかの形式で記述される。

### 基本書式

| パターン | 例 | 意味 |
|:---|:---|:---|
| `変数 = 右辺式` | `予算単価=5*X` | 指定した変数やセルに代入 |
| `= 右辺式` | `=Y+Z` | 計算結果をその行のデフォルトセル（予算数量）に返す |

### 基本ルール

- **半角ルール**: 数値、四則演算子（`+`, `-`, `*`, `/`）、比較演算子、括弧は **すべて半角** で記述する
- **ループ（収束）計算**: 値が変化しなくなるまで再計算を繰り返す
- 計算式の種類は、行属性が **計算行（Kフラグ）の計算式** とそれ以外で一部仕様が異なる
- 計算行だけで利用可能なルールには **【計算行用】** と記載

### 基本例

| 番号 | 項目名 | 予算数量 | 計算式 | 予算単価 |
|:---|:---|---:|:---|---:|
| 1001 | X | 30 | `=Y+Z` | |
| 1002 | Y | 10 | | |
| 1003 | Z | 20 | | |
| 1005 | ABC | | `予算単価=5*X` | 150 |

※ `予算単価=式` の記法は計算行以外の計算式でのみ記述可能

## 2. 変数とセルの参照方法

### 有効な変数名

- 基本数量シートにおいて、番号列に値がある変数名だけが有効（ローカル変数を除く）
- **変数名に半角スペースは使用不可**
- **基本数量シートの変数行には必ず番号を付与する**
- **変数名は重複不可**。同名が現れた場合、同一起源なら統合、そうでなければ配列等で分離

### 変数名の命名規則（⚠️ 厳守）

データ生成時には以下の命名規則を **必ず守る** こと。

- 変数名に **空白は使用不可**。発生した場合は **削除**
- 以下の半角文字は変数名に使用不可。発生した場合は **全角に変換**:
  `+ - * / = < > ! & | ^ . ( ) " ' , [ ]`
- **例外**: 半角コロン `:` および全角コロン `：` はどちらも変数名に使用不可（特殊用途のため）→ **`！`に変換** する

#### 禁止文字と変換先の対応表（データコンバート時）

| 禁止文字 | 対処 |
|:---|:---|
| 半角スペース、タブ | **削除** |
| `+ - * / = < > !` | **全角に変換**: `＋ － ＊ ／ ＝ ＜ ＞ ！` |
| `& \| ^ .` | **全角に変換**: `＆ ｜ ＾ ．` |
| `( )` | **全角に変換**: `（ ）` |
| `" '` | **全角に変換**: `＂ '` |
| `, [ ]` | **全角に変換**: `， ［ ］` |
| 半角コロン `:` | **`！`に変換**（全角コロン`：`も使用不可） |

#### `[]` サニタイズの注意

項目名に含まれる `[` `]` は全角化するが、システムが付加する配列添え字（`[0]` `[101]` `[:取引先ｺｰﾄﾞ]` 等）は半角のまま。
**サニタイズは項目名の段階で行い、配列添え字は後から付加する** ことで衝突を防ぐ。

### ローカル変数 (`@`)

- 変数名の先頭に `@` を付けると、そのシート内だけで有効な「ローカル変数」になる
- ローカル変数には番号を付与しない
- 基本数量シートのみで記述可能

### 配列変数 (`項目[添え字]`)

`@材積率[105][3]` のように、角括弧を使って配列として定義・参照が可能。

### セル変数（他行・他列の参照/代入）

| 記法 | 例 | 意味 |
|:---|:---|:---|
| `項目名:列名称` | `本体工事:見積単価` | シート内の特定セルの読み書き |
| `↑:ｾｯﾄ数` | `↑:ｾｯﾄ数` | 親行を参照 |
| `代入先=式`（コロン省略） | `予算単価=式` | 計算行以外で自行の数値列に代入する場合 |

#### 配列変数でセルを参照する場合

| 記法 | 意味 |
|:---|:---|
| `=ARRAY[IDX:予算数量]` | `=ARRAY[IDX]` と同じ意味 |
| `=ARRAY[:取引先ｺｰﾄﾞ]` | 同じ行の取引先ｺｰﾄﾞに置換して評価 |

#### 参照可能な列名

- **数値項目**: 参照可能
- **文字列項目**: `取引先名` 等の名称は参照不可。`取引先ｺｰﾄﾞ`、`原価ｺｰﾄﾞ`、`種別ｺｰﾄﾞ` を使用
- **コード自動生成時は数値のみで生成する**

### セル変数の例

| 番号 | 項目名 | 予算数量 | 計算式 | 予算単価 |
|:---|:---|---:|:---|---:|
| 1000 | IDX | 2 | | |
| 1001 | X | 20 | `=@AR[IDX]` | |
| | @AR[1] | 10 | | |
| | @AR[2] | 20 | | |
| 1002 | Y | | | 100 |
| 1005 | ABC | | `Y:予算単価=5*X` | |

※ `@AR[1]`, `@AR[2]` の記法は基本数量シートのみ記述可能
※ `Y:予算単価=5*X` の記法は計算行の計算式でのみ記述可能 **【計算行用】**

## 3. 制御と条件分岐

### IF文 【計算行用】

複数行にわたる条件分岐:
```
IF 条件
  処理
ELSE
  処理
ENDIF
```

単一行の条件分岐:
```
IF(条件) THEN 式 ELSE 式
```

### COND関数（全行で使用可能）

```
COND(条件式, 真の場合の値, 偽の場合の値)
```

入れ子にして使われることが多い:
```
=COND(延床面積<50, 1.15, COND(延床面積<70, 1.1, COND(延床面積<100, 1.05, 1.0)))
```

ただし、みつもり主任では入れ子の代わりにローカル変数のチェーンで表現するパターンが一般的:
```
@補正率A1: =COND(延床面積<50, 1.15, @補正率A2)
@補正率A2: =COND(延床面積>=50&延床面積<70, 1.1, @補正率A3)
@補正率A3: =COND(延床面積>=70&延床面積<100, 1.05, 1.0)
```

### ⚠️ COND関数での文字列比較は不可

みつもり主任には **文字列型変数が存在しない**。`COND(本体／直発注＝直発注, ...)` のような文字列比較は動作しない。

**回避策**: 条件式テキスト全体をそのまま変数名として基本数量シートに定義し、値に `0`（偽）または `1`（真）を設定する。

| 番号 | 項目名（＝変数名） | 予算数量 | 備考 |
|:---|:---|---:|:---|
| 20001 | 本体／直発注＝直発注 | 1 | ユーザーが手動設定 |

COND式からはそのまま変数名として参照される:
```
=COND(本体／直発注＝直発注, 値A, 値B)
```

## 4. 主要な関数一覧

### 算術・丸め関数

| 関数 | 書式 | 説明 |
|:---|:---|:---|
| SQRT | `SQRT(VAL)` | 平方根 |
| ROUND | `ROUND(VAL, FLAG)` | 任意の端数処理 |
| RDUP | `RDUP(VAL, FLAG)` | 切り上げ特化型 |

### 集計関数 【計算行用】

| 関数 | 説明 |
|:---|:---|
| **SUM()** | 合計範囲境界（`L`フラグ）でリセットされる。合計対象（`R`フラグ）があればその行だけを合計 |
| **TSUM()** | `R`フラグや通常の`L`フラグを無視し、シート先頭からの全有効行を累計 |

### 金額調整関数 【計算行用】

| 関数 | 書式 | 説明 |
|:---|:---|:---|
| ZEIKOMI | `ZEIKOMI(VAL, YOSAN)` | 金額指定型・税込 |
| ZEIBETU | `ZEIBETU(VAL, YOSAN)` | 金額指定型・税別 |
| RZEIKOMI | `RZEIKOMI(VAL, FLAG, FLAG2)` | 率＋丸め型・税込 |
| RZEIBETU | `RZEIBETU(VAL, FLAG, FLAG2)` | 率＋丸め型・税別 |

※ システム内部で消費税の端数を調整し、最終金額が合うように辻褄を合わせる

## 5. 計算行のフェーズとCSV特殊フォーマット

### 計算フェーズと制御コマンド 【計算行用】

計算行は1つの計算式を「予算」「見積」「プランA0C」等の各フェーズで繰り返し実行する。

| コマンド | 説明 |
|:---|:---|
| **SHEET** | フェーズの繰り返しを止め、「予算フェーズ」の1回だけ実行 |
| **LINE** | フェーズごとに代入先（予算単価、見積単価など）を自動切替しながら全フェーズ実行 |

### 【重要】CSV出力時の計算式特殊フォーマット 【計算行用】

計算行の式をCSVの「計算式」列に組み込む場合、特殊なヘッダとバイト数カウントが必要。

```
**0000:[タイトル長(4桁)]:タイトル[計算式長(4桁)]:計算式本体0000:0000:
```

#### 長さのカウント方法

- **Shift-JISエンコード時のバイト数** で指定（文字数ではない！）
- 全角 = 2バイト、半角 = 1バイト

#### 改行コードの処理

- CSVを壊さないため、式中のCRLF（改行）は実際の改行コードではなく **文字列としての `\n`（バックスラッシュ＋n: 2バイト）** に置換して出力する
