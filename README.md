## CSVフォーマット概要（クイックリファレンス）

以下は頻繁に参照する最重要事項の要約です。詳細は `docs/01_csv-format.md` を参照してください。

### ファイル基本仕様

| 項目 | 仕様 |
|:---|:---|
| 文字コード | **Shift-JIS (CP932)** |
| 改行コード | **CRLF (0x0D, 0x0A)** ─ 最終行含む全行必須 |
| 区切り文字 | カンマ (,) |
| 1行目 | 列名ヘッダ（全列をダブルクォーテーションで囲む） |
| 2行目 | 工事概要（プロジェクト情報） |
| 3行目以降 | 見積データ本体 |

### 階層構造（最重要ルール）

```
階層-0  : ルート（プロジェクト情報）─ 1件のみ
階層 1  : トップ階層（工事区分）  ─ 全て連続配置
階層-1  : 内訳開始ヘッダ        ─ 親の階層1と同じIDを使用
階層 2  : 内訳明細             ─ 新しいIDを付与、同じ親の兄弟は連続配置
階層-2  : 2階層目の内訳開始     ─ 親の階層2と同じIDを使用
階層 3  : 3階層目の明細        ─ 新しいIDを付与
（最大10階層まで）
```

**配置は「幅優先」**: 同じ親を持つ兄弟レコードは必ず連続させてから、子の詳細を配置する。

### ID管理

- **書式**: 5桁ゼロ埋め（例: `00001`）
- **ファイル内でユニーク**（ただし親子紐付けのため、階層-Nは対応する親階層Nと同じIDを持つ）

### 行属性フラグ（21桁）

各位置の文字が機能のON/OFFを制御する。不使用の位置は `0` で埋める。
**「シート」とは同じ階層の行のかたまりのこと。**

| 位置 | 文字 | スイッチ名 | 金額加算 | 印刷 | 機能概要 |
|:---|:---|:---|:---|:---|:---|
| 1 | **B** | 歩掛行 | | | 下階層の合計を単価にセットする |
| 2 | **C** | コメント行 | 禁止 | | 項目名、材質・寸法、出典根拠、備考のフィールドに跨ってコメント文字列あり |
| 3 | **T** | 小計行 | 禁止 | | シートの先頭もしくは直前の小計行の次の行から直前までの合計 |
| 4 | **N** | 値引行 | | | 値引を明示的にしているマーク |
| 5 | **H** | 非加算行 | 禁止 | | シートの合計に含めない |
| 6 | **K** | 計算行 | | | 計算式が組み込まれた行 |
| 7 | **Z** | 非課税行 | | | 消費税を計算する時に対象としない行 |
| 8 | **R** | 合計対象行 | | | SUM()関数の対象となる行。このマークが一つも無ければ先頭行から直前まで合計される |
| 9 | **L** | 合計範囲境界 | | | SUM()関数の対象となる範囲。SUM()関数より前にある合計範囲境界から直前の行までが合計対象 |
| 10 | **X** | 非印刷行 | 禁止 | 不可 | 加算も印刷もしない行。見積もり作成者のコメントとか入る |
| 11 | **M** | 非加算計算行記憶 | 禁止 | | 元々非加算計算行であったことの記憶。非加算計算行は計算行と非加算行の合わせ属性 |
| 12 | **S** | 基本数量行 | 禁止 | 不可 | この下階層は番号列または名称列をキーにした値（予算数量）を格納した行の集まり。「計算式」列による計算結果も格納される |
| 13 | **Y** | 消費税行 | 禁止 | | 直前までの合計から計算した消費税の金額 |
| 14 | **I** | 印刷明細行 | | | この下階層は印刷しない。時々、下階層が無くても強制印刷を行う目的で設定される場合もある |
| 15 | **A** | 請求行 | 禁止 | 不可 | 請求書を作成する時の今回請求額または入金済額が設定された行 |
| 16 | **F** | 数量計算式有 | | | その行の数量には計算式が設定されている。「計算式」列ではない別の所に格納された計算式 |
| 17 | **F** | 単価計算式有 | | | その行の単価には計算式が設定されている。「計算式」列ではない別の所に格納された計算式 |
| 18 | **F** | 単位量計算式有 | | | その行の単位量には計算式が設定されている。「計算式」列ではない別の所に格納された計算式 |
| 19 | **D** | 改行マーク | | | 印刷時にこの行の次で改ページする |
| 20 | **I** | 変更不可マーク | | | シート入力時に変更できない列 |
| 21 | **O** | 展開印刷行 | | | 印刷時にこの行の直後に下階層を続けて印刷する。ただし、直下の1階層だけ印刷 |

#### コメント行（Cフラグ）の特殊ルール

- **1行コメント**: 「番号」列に **` C`（半角スペース1つ＋大文字C）** を必ず入力
- **複数行コメント**: 「番号」列に **`L5`** 等（`L`＋印刷時の行数）を入力。`L5`は5行分のコメントエリアを確保

#### フラグの組み合わせ例

| 行属性文字列 | 意味 |
|:---|:---|
| `000000000000000000000` | 通常の明細行（全フラグOFF） |
| `B000H0000X0S000000000` | 基本数量シートの親行（歩掛+非加算+非印刷+基本数量） |
| `00000K000000000000000` | 計算行 |
| `0C000000000000000000` | コメント行 |

### 引用符ルール

3行目以降のデータ行において:
- **ダブルクォーテーションで囲む列**: 番号, 項目名, 材質・寸法, 予算単位, 見積単位, 備考, 取引先ｺｰﾄﾞ, 取引先名, 原価ｺｰﾄﾞ, 原価名, 種別ｺｰﾄﾞ, 種別名, 出典根拠, 分類名, 部分名, 計算式, 歩掛/単位
- **囲まない列**: 階層, ＩＤ, 行属性, 予算数量, 予算単価, 見積数量, 区分, 掛率 等の数値・フラグ列

### 列一覧（ヘッダ行）

```
"階層","ＩＤ","行属性","原価区分","番号","項目名","材質・寸法","予算数量","予算単位","予算単価","予算金額","見積数量","見積単位","見積単価","見積金額","掛率","備考","取引先ｺｰﾄﾞ","取引先名","区分","原価ｺｰﾄﾞ","原価名","種別ｺｰﾄﾞ","種別名","出典根拠","分類名","部分名","計算式","積算１","積算２","単体数量","単位量","歩掛/単位","数量/端数処理","単価/端数処理","金額/端数処理","雑品/処理区分"
```

## 笞ꀀ・輀 AIが間違えやすいポイント（必読）

以下はAIがデータ生成・編集時に特に間違えやすいポイントです。**出力前に必ず確認してください。**

### 致命的エラー（データが壊れる）

1. **改行コードをLFにしてしまう**
   - 笨需 LF (`0x0A`) のみ → インポートエンジンが行を認識できない
   - 笨錀 必ず CRLF (`0x0D, 0x0A`)。最終行にもCRLFが必要

2. **文字コードをUTF-8で出力してしまう**
   - 笨需 UTF-8 → 日本語が文字化け
   - 笨錀 必ず Shift-JIS (CP932) で出力

3. **階層-1のIDを新規採番してしまう**
   - 笨需 `-1,00099,...` （新しいID）
   - 笨錀 `-1,00002,...` （対応する階層1のIDと同じ）

4. **同じ親の兄弟レコードが連続していない**
   - 笨需 階層2のレコードAの後に階層-2を挟んで、同じ親のレコードBを配置
   - 笨錀 レコードA、レコードBを連続配置してから、階層-2以降を配置（幅優先）

### 重大エラー（計算が狂う）

5. **基本数量シートの変数に番号を付け忘れる**
   - ローカル変数（@始まり）以外の変数行には必ず番号列に値を設定する

6. **変数名に半角演算子を含めてしまう**
   - 笨需 `1F床面積+2F床面積` （変数名に+が入っている）
   - 笨錀 `1F床面積＋2F床面積` （全角に変換）、または別名を使用
   - 使用禁止文字: `+ - * / = < > ! & | ^ . ( ) " ' , [ ]` および半角・全角コロン
   - **特に注意**: 半角コロン `:` → 全角コロン `：` も使用不可 → **`！`に変換** する
   - **特に注意**: `[` `]` のサニタイズは **項目名の段階で全角化** し、配列添え字 `[0]` `[101]` 等は後から半角で付加する

7. **COND式で文字列比較をしてしまう**
   - 笨需 `COND(本体／直発注＝直発注, ...)` （文字列比較は動作しない）
   - 笨錀 条件式テキスト自体を変数名として基本数量シートに定義し、値に 0 or 1 を設定
   - 例: 変数名 `本体／直発注＝直発注`、値 `1`（ユーザーが手動設定）→ COND式からそのまま参照

8. **計算式内の数値や演算子を全角で書いてしまう**
   - 笨需 `＝延床面積＊０．３０２５`
   - 笨錀 `=延床面積*0.3025`

9. **Shift-JISバイト数を文字数で数えてしまう（計算行のCSV特殊フォーマット）**
   - 計算行の長さ指定は **Shift-JISエンコード時のバイト数**（全角=2バイト、半角=1バイト）
   - 文字数ではなくバイト数であることに注意

### 軽微だが注意が必要なエラー

10. **コメント行の番号列を空欄にしてしまう**
   - 1行コメント → 番号列に ` C`（半角スペース1つ＋大文字C）
   - 複数行コメント → 番号列に `L5` 等（Lと行数）

11. **テンプレートの`[]`（空括弧）を置換し忘れる**
    - 子階層の計算式に `[]` が残っている場合、親階層の識別子で置換が必要

## 出力前チェックリスト

CSVデータを生成・編集した後、以下を順番に確認してください。

```
□ 1. 文字コードは Shift-JIS (CP932) か
□ 2. 改行コードは CRLF か（最終行含む）
□ 3. 1行目のヘッダは全列がダブルクォーテーションで囲まれているか
□ 4. 2行目は工事概要行（階層-0）か
□ 5. 階層1のレコードは全て連続して配置されているか
□ 6. 階層-Nは対応する階層Nと同じIDを持っているか
□ 7. 同じ親を持つ兄弟レコードは連続配置されているか（幅優先）
□ 8. IDは5桁ゼロ埋めでファイル内ユニークか
□ 9. 行属性は21桁か（不要な位置は '0' で埋めているか）
□ 10. 文字列列はダブルクォーテーションで囲み、数値列は囲んでいないか
□ 11. 基本数量シートの変数行（@以外）に番号が付与されているか
□ 12. 変数名に禁止文字（半角演算子・コロン等）が含まれていないか
□ 13. 変数名中のコロンは `！` に変換されているか（全角コロンも不可）
□ 14. 変数名中の `[]` は全角化済みか（配列添え字と区別できているか）
□ 15. 計算式内の数値・演算子は全て半角か
□ 16. COND式で文字列比較をしていないか（変数名として定義する方式か）
□ 17. テンプレートの [] が適切に置換されているか（またはテンプレートとして残す意図があるか）
```
